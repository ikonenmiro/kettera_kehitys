- Mitä ketterällä kehityksellä tarkoitetaan?

Ketterä kehitys on ohjelmistokehys tapa, jossa työskennellään iteraktiivisesti ja joustavasti, tiiviissä yhteystyssä asiakkaan kanssa. Painopiste on nopeassa palautteessa, toimivissa ratkaisuissa ja muutoksiin sopeutumisessa suunnitelmien 

- Mitä menetelmää käytettiin ennen ketterää kehitystä?

Ennen ketterää kehitystä yleensä käytetään vesputous mallia, jossa kehitys etenee vaiheittain (suunnittelu → toteutus → testaus → käyttöönotto) ilman suuria muutoksia

- Miksi ketterään kehitykseen on siirrytty?

Ketterään kehitykseen siirryttiin pääasiassa siksi, että perinteinen vesiputousmalli oli liian hidas ja jäykkä

lyhyesti syyt:
Asiakkaan tarpeet muuttuvat nopeasti
Markkinat ja teknologia kehittyvät hurjaa vauhtia
Vesiputousmallissa vasta lopussa selviää, toimiiko tuote oikeasti
Haluttiin nopeampaa palautetta, pienempiä riskejä ja mahdollisuus reagoida muutoksiin

→ Ketterä = nopeampi oppiminen + vähemmän turhaa työtä väärään suuntaan.

- Kerro lyhyesti Agilesta sekä mitä siinä määritellään?

Agile on ketterän kehityksen periaatteisto, joka korostaa joustavuutta, yhteistyötä ja jatkuvaa parantamista.
Siinä määritellään 4 arvoa ja 12 periaatetta, jotka ohjaavat tiimien toimintaa ohjelmistokehityksessä.

- Miten Scrumin ja Kanbanin olennainen ero?

Scrum on rytmikäs, aikalaatikoitu työskentely → sopii hyvin, kun halutaan säännöllistä palautetta ja selkeää etenemistä

Kanban on joustava, visuaalinen työnvirran hallinta → sopii, kun prioriteetit muuttuvat usein tai työ on jatkuvaa virtaa

- Miten Jira liittyy ketterään kehitykseen?

Jira on työkalu, jota käytetään ketterässä kehityksessä työn suunnitteluun, seurantaan ja hallintaan.
Sillä hallitaan esimerkiksi backlogia, sprinttejä, tehtäviä ja virheitä sekä seurataan tiimin edistymistä Agile-menetelmien, kuten Scrumin tai Kanbanin, mukaisesti.

- Miten Git edistää ketterää kehitystä?

Git edistää ketterää kehitystä näin lyhyesti:

Mahdollistaa nopeat kokeilut ja pienet muutokset (branchit)
Tukee jatkettavaa kehitystä ilman pelkoa rikkomisesta
Helpottaa usean kehittäjän samanaikaista työskentelyä (feature branchit, pull requestit)
Tarjoaa selkeän historian ja helpon rollbackin
Nopeuttaa feedback-looppia (ci/cd + branch-per-feature)

→ Git tekee ketterästä kehityksestä käytännössä mahdollista.
